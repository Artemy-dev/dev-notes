### Структура HTTP протокола

**HTTP запрос (POST с телом)**

```bash
POST /api/login HTTP/1.1
Host: example.com
Content-Type: application/x-www-form-urlencoded
Content-Length: 27
User-Agent: Mozilla/5.0

username=admin&password=123456
```

**Разбор запроса**

```text
POST /api/login HTTP/1.1
│    │            └─── Версия HTTP
│    └─── Путь к ресурсу
└─── Метод запроса

Host: example.com
└─── Доменное имя или IP-адрес сервера, к которому направлен запрос

Content-Type: application/x-www-form-urlencoded
│             └─── Конкретный способ: пары "ключ=значение" через & с URL-кодированием
└─── Формат и способ кодирования данных, передаваемых в теле запроса

Content-Length: 27
│               └─── Длина тела = 27 байт
└─── Размер тела запроса в байтах

User-Agent: Mozilla/5.0
└─── Идентификационная строка клиентского приложения, содержащая информацию о браузере, ОС и движке

username=admin&password=123456
└─── Тело HTTP-запроса (body)
```

### HTTP-методы

**GET** - запросить данные. Только чтение, ничего не меняет на сервере. Данные передаются в URL (видны всем).
Пример: открыть страницу, получить список товаров.

```bash
GET /api/users HTTP/1.1
Host: example.com
```

**POST** - отправить данные на сервер для создания чего-то нового. Данные передаются в теле запроса (скрыты).
Пример: регистрация, вход в аккаунт, отправка комментария.

```bash
POST /api/users HTTP/1.1
Host: example.com
Content-Type: application/json

{"name": "Иван", "email": "ivan@mail.ru"}
```

**PUT** - полностью заменить существующий ресурс новыми данными.
Пример: обновить всю анкету пользователя целиком.

```bash
PUT /api/users/42 HTTP/1.1
Host: example.com
Content-Type: application/json

{"name": "Иван", "email": "new@mail.ru", "age": 30}
```

**PATCH** - частично обновить ресурс (только указанные поля).
Пример: сменить только email, не трогая остальные данные профиля.

```bash
PATCH /api/users/42 HTTP/1.1
Host: example.com
Content-Type: application/json

{"email": "new@mail.ru"}
```

**DELETE** - удалить ресурс.
Пример: удалить пост, аккаунт, товар.

```bash
DELETE /api/users/42 HTTP/1.1
Host: example.com
```

**HEAD** - то же, что GET, но сервер возвращает только заголовки без тела.
Пример: проверить, существует ли файл, не скачивая его.

```bash
HEAD /api/file.zip HTTP/1.1
Host: example.com
```

**OPTIONS** - узнать, какие методы разрешены для ресурса.
Пример: браузер проверяет перед сложным запросом (CORS).

```bash
OPTIONS /api/users HTTP/1.1
Host: example.com
```

**TRACE** - возвращает запрос обратно (для отладки, редко используется).

```bash
TRACE /api/test HTTP/1.1
Host: example.com
```

### Клиент-серверная архитектура

**Клиент** - программа (или устройство), которая отправляет запрос и хочет получить данные.
Пример: твой браузер, мобильное приложение или Postman.

**Сервер** - программа (или мощный компьютер), которая принимает запрос, обрабатывает его и отправляет ответ.
Пример: компьютер, на котором крутится сайт или API.

**Ресурсы** - данные или объекты, к которым обращается клиент. Каждый ресурс имеет свой адрес (URL).
Пример: страница пользователя, список товаров, фотография.

**Веб-серверные приложения (веб-серверы)** - программы, которые работают на сервере и отвечают за приём HTTP-запросов от клиентов и отправку ответов (например, HTML-страниц).

**Популярные веб-серверы:**

* **Nginx** - Самый популярный веб-сервер. Асинхронная архитектура позволяет держать много соединений, отлично справляется с высокой нагрузкой без просадок по скорости.
* **Cloudflare Server** - Глобальная сеть доставки контента (CDN). Работает как обратный прокси-сервер: твой сайт физически на одном сервере, а Cloudflare кеширует его копии по всему миру, ускоряет загрузку, заодно защищая от DDoS-атак.
* **Apache** - Модульная система: можно подключать только нужные функции. Главная особенность - файлы `.htaccess`, которые позволяют менять конфигурацию сервера "на лету" для каждой папки без перезапуска.
**LiteSpeed** - Замена **Apache**, почти полностью с ним совместима, но работает значительно быстрее. Внутри встроенный кеш для **WordPress** и других **CMS** (инструмент, который делает создание и ведение сайта доступным для любого человека, а не только для программистов), что позволяет обходиться без лишних плагинов и сильно ускоряет сайты.
* **Node.js** - Среда для запуска **JavaScript** вне браузера. Внутри есть встроенный HTTP-модуль, который позволяет писать серверную часть приложений на **JavaScript** - ключевой момент в стеке, где фронтенд и бэкенд на одном языке.

### Классификатор уязвимостей OWASP (Open Web Application Security Project)

**OWASP** - глобальный стандарт и список самых критичных угроз для веб-приложений, обновляется раз в несколько лет на основе реальных данных об атаках.

* **A01: Broken Access Control (Нарушение контроля доступа)**.
Пользователь получает доступ к чужим данным или функциям, которые должны быть ему запрещены.
Например, меняет ID в URL и видит чужой профиль.
* **A02: Security Misconfiguration (Ошибки конфигурации безопасности)**.
Стандартные пароли не меняются, отладочный режим включен, лишние порты открыты.
Приложения выдают слишком подробные ошибки, раскрывая внутреннее устройство.
* **A03: Software Supply Chain Failures (Компрометация цепочки поставок ПО)**.
Атакующий взламывает популярную библиотеку, плагин или процесс сборки.
Зловредный код автоматически попадает во все приложения, которые это используют.
* **A04: Cryptographic Failures (Криптографические ошибки)**.
Пароли хранятся без хеширования, данные передаются по HTTP без шифрования, используются старые алгоритмы вроде MD5.
* **A05: Injection (Инъекции)**.
Пользователь вводит вредоносный код, который сервер принимает за команду.
SQL-инъекция позволяет украсть всю базу, XSS - скрипт в браузере жертвы.
* **A06: Insecure Design (Небезопасное проектирование)**.
Ошибки на уровне архитектуры, которые нельзя исправить одним патчем. Например, восстановление пароля без проверки кода и ограничения попыток.
* **A07: Identification and Authentication Failures (Ошибки идентификации и аутентификации)**.
Слабые пароли разрешены, сессии не сбрасываются при выходе, двухфакторная аутентификация отсутствует.
Атакующий подбирает учетные данные и входит в чужой аккаунт.
* **A08: Software and Data Integrity Failures (Нарушения целостности данных)**.
Приложение не проверяет цифровую подпись обновлений или распаковывает (десериализует) данные из ненадежного источника, позволяя выполнить произвольный код. 
Пример: Атакующий вместо простого JSON отправляет специально сформированный сериализованный объект, который заставляет библиотеку десериализации вызвать опасные конструкторы, геттеры или методы.
* **A09: Security Logging and Monitoring Failures (Ошибки журналирования и мониторинга)**.
Атаки не записываются в логи, SIEM не настроен, оповещения не приходят.
Взлом остается незамеченным месяцами.
* **A10: Mishandling of Exceptional Conditions (Неверная обработка исключительных ситуаций)**.
Ошибки обрабатываются так, что система раскрывает внутренние данные или переходит в небезопасное состояние.
Например, сбой проверки прав доступа открывает запрещенную функцию.
